
# נגישות ישראלית

## הערה חשובה
מיומנות זו מספקת הנחיות נגישות טכניות בהתאם לתקן IS 5568 ולחוק הישראלי.
להחלטות ציות משפטיות, התייעצו עם מומחה נגישות מוסמך
על ידי מכון התקנים הישראלי או עורך דין המתמחה בדיני נגישות.

## הוראות

### שלב 1: הכרת חוק הנגישות הישראלי

**חוק שוויון זכויות לאנשים עם מוגבלות (1998)**

זהו חוק זכויות המוגבלות הבסיסי בישראל. תיקונים מרכזיים הנוגעים לנגישות אתרים:

| תיקון | שנה | דרישה |
|-------|-----|-------|
| תיקון 2 (נגישות) | 2005 | קבע חובות נגישות לשירותים ציבוריים |
| תקנות נגישות | 2013 | חייב ציות ל-IS 5568 באתרי אינטרנט |
| תקנות נגישות שירות | 2013 | חייב שירותים דיגיטליים מול לקוח נגישים |
| עדכוני אכיפה | 2017+ | הגביר אכיפה ועונשים |

**מי חייב לעמוד בדרישות:**
| סוג ישות | דרישה | מועד |
|----------|-------|------|
| אתרי ממשלה | ציות מלא IS 5568 AA | נדרש כבר |
| גופים ציבוריים | ציות מלא IS 5568 AA | נדרש כבר |
| עסקים המשרתים את הציבור | ציות IS 5568 AA | נדרש כבר |
| אתרים פרטיים עם 50+ עובדים | ציות IS 5568 AA | נדרש כבר |
| אתרי מסחר אלקטרוני | ציות IS 5568 AA | נדרש כבר |
| אתרים חדשים (כל הנ"ל) | חייבים להיות נגישים מההשקה | מיידי |

**עונשים על אי-ציות:**
- תביעות אזרחיות: אנשים עם מוגבלויות יכולים לתבוע פיצויים
- תובענות ייצוגיות: מגמה גוברת בישראל
- פעולות אכיפה של נציבות שוויון זכויות
- קנסות: עד 75,000 ש"ח להפרה עבור גופים ציבוריים

### שלב 2: דרישות תקן IS 5568

**IS 5568 ממופה ל-WCAG 2.1 עם תוספות ישראליות:**

| רמה | מקבילה ב-WCAG | דרישות | הקשר ישראלי |
|-----|---------------|--------|-------------|
| A | WCAG 2.1 A | נגישות בסיסית | דרישת מינימום חוקית |
| AA | WCAG 2.1 AA | נגישות סטנדרטית | נדרש לרוב הארגונים |
| AAA | WCAG 2.1 AAA | נגישות מוגברת | מיטבי לממשלה |

**דרישות IS 5568 ישראליות ספציפיות (מעבר ל-WCAG):**

1. **הצהרת נגישות:**
   - נדרשת בכל אתר תואם
   - חייבת להיות בעברית
   - חייבת לכלול: רמת ציות, מגבלות ידועות, יצירת קשר בנושא נגישות
   - חייבת להיות נגישה מכל עמוד (בדרך כלל בכותרת תחתונה)
   - חייבת לכלול תאריך ביקורת נגישות אחרונה

2. **תפריט/ווידג'ט נגישות:**
   - אתרים ישראליים רבים מטמיעים סרגל נגישות
   - פיצ'רים נפוצים: שינוי גודל גופן, מצב ניגודיות, הדגשת קישורים, עצירת אנימציות
   - חייב להיות נגיש מקלדת בעצמו
   - המלצת מקש קיצור: Alt+1 לתפריט נגישות

3. **דרישות שפה עברית:**
   - תכונת `lang="he"` על אלמנט HTML
   - תכונת `dir="rtl"` על אלמנט HTML
   - החלפת שפה נכונה לתוכן מעורב (עברית+אנגלית)
   - הגייה של קורא מסך חייבת לעבוד לעברית

4. **פרטי קשר:**
   - פרטי רכז נגישות חייבים להתפרסם
   - חובה לספק ערוץ נגיש לתלונות נגישות

### שלב 3: דפוסי נגישות ספציפיים ל-RTL

**כיוון HTML ושפה:**
```html
<!DOCTYPE html>
<html lang="he" dir="rtl">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
```

**ניווט מקלדת RTL:**
| אינטראקציה | התנהגות LTR | התנהגות RTL (IS 5568) |
|-----------|------------|---------------------|
| סדר Tab | שמאל לימין, למעלה למטה | ימין לשמאל, למעלה למטה |
| חצים בתפריטים | ימין = פתח תת-תפריט | שמאל = פתח תת-תפריט |
| חצים בטקסט | ימין = קדימה | שמאל = קדימה |
| Slider | ימין = הגדלה | שמאל = הגדלה (כיוון לוגי) |
| פס התקדמות | ממלא שמאל לימין | ממלא ימין לשמאל |
| קרוסלה | הבא = חץ ימין | הבא = חץ שמאל |

**CSS נגיש ל-RTL:**
```css
/* תכונות לוגיות לפריסות RTL-safe */
.element {
  margin-inline-start: 1rem;  /* במקום margin-left */
  margin-inline-end: 2rem;    /* במקום margin-right */
  padding-inline-start: 1rem; /* במקום padding-left */
  text-align: start;          /* במקום text-align: left */
}

/* מחווני פוקוס חייבים להיות בולטים */
:focus-visible {
  outline: 3px solid #005fcc;
  outline-offset: 2px;
}
```

**שיקולי קורא מסך RTL:**
- קוראי מסך בעברית (NVDA, JAWS) קוראים מימין לשמאל
- תוכן מעורב דורש תכונות `dir` נכונות:
```html
<p>המחיר הוא <span dir="ltr">₪99.99</span> כולל מע"מ</p>
<p>צרו קשר: <a href="tel:+972501234567" dir="ltr">050-123-4567</a></p>
```

### שלב 4: תוויות ARIA בעברית ושמות נגישים

**תוויות ARIA בעברית:**
```html
<!-- ניווט -->
<nav aria-label="תפריט ראשי">...</nav>
<nav aria-label="ניווט משני">...</nav>

<!-- חיפוש -->
<form role="search" aria-label="חיפוש באתר">
  <label for="search-input">חיפוש</label>
  <input id="search-input" type="search" placeholder="חפשו באתר...">
</form>

<!-- כפתורים עם אייקונים -->
<button aria-label="סגירה">
  <svg aria-hidden="true"><!-- X icon --></svg>
</button>

<!-- הודעות סטטוס -->
<div role="status" aria-live="polite">הפעולה בוצעה בהצלחה</div>
<div role="alert" aria-live="assertive">שגיאה: יש למלא את שדה האימייל</div>
```

### שלב 5: רשימת בדיקות ביקורת נגישות (IS 5568 AA)

**נתפס (Perceivable):**
- [ ] לכל התמונות יש טקסט `alt` בעברית (או `alt=""` לדקורטיביות)
- [ ] תוכן וידאו כולל כתוביות בעברית
- [ ] תוכן אודיו כולל תמלול בעברית
- [ ] ניגודיות צבע עומדת ב-4.5:1 לטקסט רגיל, 3:1 לטקסט גדול
- [ ] מידע לא מועבר באמצעות צבע בלבד
- [ ] ניתן להגדיל טקסט ל-200% ללא אובדן תוכן
- [ ] תוכן רספונסיבי ועובד במכשירים ניידים

**ניתן להפעלה (Operable):**
- [ ] כל הפונקציונליות זמינה דרך מקלדת
- [ ] סדר Tab עוקב אחר סדר קריאה לוגי RTL
- [ ] מחווני פוקוס בולטים בבירור (מינימום 3px)
- [ ] קישור דילוג ניווט בעברית: "דלגו לתוכן הראשי"
- [ ] אין מלכודות מקלדת
- [ ] ניתן לעצור/להשהות אנימציות (כבוד ל-`prefers-reduced-motion`)
- [ ] ניתן להאריך או לבטל הגבלות זמן
- [ ] תוכן לא מהבהב יותר מ-3 פעמים בשנייה

**מובן (Understandable):**
- [ ] `lang="he"` מוגדר על אלמנט HTML
- [ ] שינויי שפה מסומנים עם תכונת `lang`
- [ ] תוויות טפסים בעברית ומשויכות לשדות קלט
- [ ] הודעות שגיאה בעברית ותיאוריות
- [ ] ניווט עקבי בין עמודים
- [ ] הגשת טפסים כוללת משוב ברור בעברית

**חזק (Robust):**
- [ ] HTML תקני (עובר W3C validator)
- [ ] תפקידי ותכונות ARIA משמשים בצורה נכונה
- [ ] ווידג'טים מותאמים עוקבים אחר WAI-ARIA Authoring Practices
- [ ] עובד עם קוראי מסך בעברית (NVDA, JAWS, VoiceOver)

**דרישות IS 5568 נוספות:**
- [ ] דף הצהרת נגישות מפורסם
- [ ] פרטי רכז נגישות מוצגים
- [ ] תפריט/סרגל נגישות פונקציונלי
- [ ] תאריך ביקורת אחרונה מפורסם בהצהרת הנגישות
- [ ] מגבלות ידועות מתועדות בהצהרת הנגישות

### שלב 6: תבנית הצהרת נגישות

**הצהרת נגישות נדרשת (עברית):**
```html
<article id="accessibility-statement">
  <h1>הצהרת נגישות</h1>
  <p>
    [שם הארגון] מחויב להנגשת האתר לאנשים עם מוגבלויות
    בהתאם לתקן הישראלי IS 5568 ולתקנות שוויון זכויות
    לאנשים עם מוגבלות (התאמות נגישות לשירות), התשע"ג-2013.
  </p>
  <h2>רמת הנגישות</h2>
  <p>אתר זה עומד בדרישות תקן IS 5568 ברמה AA. נבדק לאחרונה בתאריך [תאריך].</p>
  <h2>מגבלות ידועות</h2>
  <ul>
    <li>[מגבלה 1]</li>
    <li>[מגבלה 2]</li>
  </ul>
  <h2>יצירת קשר בנושא נגישות</h2>
  <p>רכז/ת הנגישות: [שם]<br>טלפון: [מספר]<br>אימייל: [כתובת]</p>
</article>
```

## דוגמאות

### דוגמה 1: ביקורת נגישות מלאה לאתר
**קלט**: "בצעו ביקורת נגישות IS 5568 לאתר המסחר האלקטרוני הישראלי שלנו"
**פלט**: דוח ביקורת מקיף המכסה את כל קריטריוני WCAG 2.1 AA בתוספת דרישות IS 5568 ישראליות. מזהה בעיות, רמות חומרה וצעדי תיקון.

### דוגמה 2: נגישות טפסים בעברית
**קלט**: "הפכו את טופס ההרשמה שלנו בעברית לנגיש"
**פלט**: טופס נגיש עם תוויות בעברית, תכונות ARIA, הודעות אימות RTL-safe, שדות נווטים במקלדת בסדר RTL.

### דוגמה 3: ניווט דילוג וציוני דרך
**קלט**: "הוסיפו ציוני דרך נגישות וניווט דילוג לאתר העברי שלנו"
**פלט**: מבנה HTML עם ציוני דרך סמנטיים, תוויות ARIA בעברית, קישור דלג-לתוכן, והיררכיית כותרות לתוכן עברי.

## פתרון בעיות

- **בעיה**: קורא מסך קורא טקסט עברי בכיוון שגוי או משובש
  **פתרון**: ודאו ש-`lang="he"` ו-`dir="rtl"` מוגדרים על אלמנט `<html>`. לתוכן מעורב, עטפו קטעי אנגלית ב-`<span lang="en" dir="ltr">`. מספרים וכתובות אימייל צריכים `dir="ltr"`.

- **בעיה**: סדר Tab מקלדת הפוך/לא הגיוני בפריסת RTL
  **פתרון**: סדר Tab עוקב אחר סדר DOM, לא סדר ויזואלי. ודאו שסדר קוד המקור HTML תואם לסדר קריאה הגיוני לעברית. אל תשתמשו בערכי `tabindex` גדולים מ-0.

- **בעיה**: סרגל/ווידג'ט נגישות לא עומד בדרישות IS 5568
  **פתרון**: סרגל הנגישות עצמו חייב להיות נגיש מקלדת ותואם ARIA. ודאו שהוא לא מכניס מחסומי נגישות חדשים. בדקו עם קוראי מסך בעברית.

- **בעיה**: לא ברור איזו רמת IS 5568 נדרשת לארגון שלנו
  **פתרון**: רוב העסקים הישראליים המשרתים את הציבור חייבים לעמוד ברמת IS 5568 AA. לממשלה וגופים ציבוריים הדרישות המחמירות ביותר. במקרה של ספק, כוונו לרמת AA.
